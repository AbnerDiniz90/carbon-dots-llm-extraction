Your objective is to compare a candidate JSON extraction against a reference (ground truth) JSON, adhering to strict scientific and data integrity principles.

**Input Structure:**

*   **Block A (Learning Context):**
    *   `[[Json_Example]]`: A historical example of a correctly extracted JSON.
    *   `[[Expert_Label]]`: The expert's judgment on the `[[Json_Example]]` (Good/Bad).
    *   `[[Expert_Comment]]`: The expert's reasoning for the `[[Expert_Label]]` on the `[[Json_Example]]`.
    *   **Purpose:** Analyze this block to calibrate your understanding of acceptable variations, common errors, and the expert's tolerance for different types of discrepancies.

*   **Block B (Current Task):**
    *   `[[Json_Reference]]`: The ground truth JSON, manually extracted by an expert. This is your benchmark.
    *   `[[Json_Candidate]]`: The JSON extracted by an LLM. This is the data you will evaluate.

**Evaluation Criteria and Reasoning Logic:**

Your evaluation must distinguish between **Good** (accurate, semantically equivalent, or valid enrichment) and **Bad** (factually incorrect, hallucinated, or omitted critical data).

1.  **Factual Accuracy (Quantitative):**
    *   **Rule:** Numerical values (e.g., `[[value]] nm`, `[[value]]%`, `[[value]] s`) and their associated units MUST match the `[[Json_Reference]]` exactly.
    *   **Strict Tolerance:** A scientifically negligible margin is permissible ONLY if it is a standard deviation or error range explicitly stated in the reference. Otherwise, any numerical discrepancy is a **Critical Error**.
    *   **Example:** If `[[Json_Reference]]` states `quantum_yield.value: 50%` and `[[Json_Candidate]]` states `quantum_yield.value: 90%`, this is a **Bad** evaluation.

2.  **Descriptive Accuracy (Qualitative - FLEXIBLE):**
    *   **Information Gain (Enrichment):** If `[[Json_Candidate]]` includes valid chemical adjectives or functional descriptions not present in `[[Json_Reference]]` (e.g., Reference: `definition: "Carbon Dots"`; Candidate: `definition: "Fluorescent Carbon Dots with peroxidase activity"`), this is considered **Good/Enrichment**, provided it does not explicitly contradict the `[[Json_Reference]]`. This is treated as a discovery or a more detailed description.
    *   **Semantic Equivalence:** Synonymous terms or phrases are considered equivalent.
        *   `N-doped` is equivalent to `Nitrogen-doped`.
        *   `Hydrothermal` is equivalent to `Hydrothermal synthesis`.
    *   **Rule:** If terms are semantically equivalent and do not introduce factual contradictions, the extraction is **Good**.

3.  **Hallucination Check (Fabrication):**
    *   **PENALIZE ONLY IF:**
        *   `[[Json_Candidate]]` invents **Quantitative Data** (numbers, formulas) that are missing (`null`) in `[[Json_Reference]]`.
        *   `[[Json_Candidate]]` adds descriptions that imply a fundamentally different material or property not supported by the reference (e.g., stating `morphology: "Graphene Oxide sheets"` when the reference is about Carbon Dots).
    *   **DO NOT PENALIZE:** If `[[Json_Candidate]]` fills a `null` field in `[[Json_Reference]]` with scientifically plausible data that is not a direct contradiction and could be considered a valid discovery or inference from the source text (this is **Discovery**, a form of **Good** if not factually wrong).

4.  **Omission Check:**
    *   **PENALIZE IF:** `[[Json_Candidate]]` has a `null` value for a field where `[[Json_Reference]]` contains specific, non-null data. This indicates missing information.

**Analysis Steps:**

*   **Step 1:** Carefully analyze Block A (`[[Json_Example]]`, `[[Expert_Label]]`, `[[Expert_Comment]]`) to understand the expert's judgment patterns, acceptable deviations, and the definition of "Good" and "Bad" in practice.
*   **Step 2:** Systematically compare `[[Json_Reference]]` and `[[Json_Candidate]]` field by field, applying the criteria above.

    *   Note any missing data where the reference has information as **Omissions** (**Bad**).
*   **Step 3:** Determine the overall quality of `[[Json_Candidate]]` based on the severity and number of discrepancies.
    *   A single critical factual error or significant hallucination results in a **Bad** evaluation.
    *   Minor semantic differences or valid enrichments contribute to a **Good** evaluation, even with minor field differences.
