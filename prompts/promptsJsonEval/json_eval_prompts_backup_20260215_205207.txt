This is distinct from outright fabrication.

4.  **Omission Check (Missing Information - PENALIZE IF):**
    *   **PENALIZE IF:** The `[[Json_Candidate]]` has a `null` value for a field where the `[[Json_Reference]]` contains specific, non-null data. This indicates a failure to extract crucial information.

**Block B: Current Task (Evaluation)**

You will be presented with two JSON objects: `[[Json_Reference]]` (the Ground Truth) and `[[Json_Candidate]]` (the LLM's extraction). Your task is to compare the `[[Json_Candidate]]` against the `[[Json_Reference]]` and assign a label of "Good" or "Bad" based on the following criteria:

**Evaluation Criteria:**

1.  **Factual Accuracy (Quantitative):**
    *   Numerical values (e.g., `[[size.avg]]`, `[[quantum_yield.value]]`, `[[elemental_composition.C]]`) and their corresponding units MUST match the Reference JSON exactly, or be within a scientifically negligible margin of error.
    *   *Strict Rule:* If the Reference JSON contains a specific numerical value (e.g., `[[quantum_yield.value]]: 50%`), and the Candidate JSON reports a different value (e.g., `[[quantum_yield.value]]: 90%`), this constitutes a Critical Error and the Candidate is "Bad".

2.  **Descriptive Accuracy (Qualitative - FLEXIBLE):**
    *   **Information Gain (Enrichment):** If the Candidate JSON includes valid chemical adjectives, functional descriptions, or synthesis details NOT present in the Reference JSON (e.g., Reference: `[[carbon_dots.general_info.definition]]: Carbon Dots`; Candidate: `[[carbon_dots.general_info.definition]]: Fluorescent Carbon Dots with peroxidase activity`), treat this as CORRECT/ENRICHMENT, and label as "Good", *provided* the added information does not explicitly contradict the Reference JSON.
    *   **Semantic Equivalence:** Consider semantically equivalent phrases as matches. For example, "N-doped" is equivalent to "Nitrogen-doped", and "Hydrothermal" is equivalent to "Hydrothermal synthesis".

    *   **DO NOT PENALIZE:** If the Candidate JSON fills a `null` field in the Reference JSON with scientifically plausible data supported by the context of the research paper (treat this as Discovery and label as "Good").

4.  **Omission Check:**
    *   Penalize if the Candidate JSON has a `null` value where the Reference JSON contains specific data. This indicates a failure to extract information. In this case, label as "Bad".

**Definitions:**

*   **Bad:** The Candidate JSON contains significant factual errors, contradictions, or omissions compared to the Reference JSON. Hallucination of quantitative data or misrepresentation of the material is also grounds for a "Bad" label.
